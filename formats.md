# 파일 포맷 명세

## ISO 레벨

- ISO9660, 133개 파일. 실질 데이터는 `/DATA.BIN`(약 1.27GB) 하나에 집중.
- `DATA.BIN`의 ISO 내 시작 오프셋 = **LBA 2117 × 2048 = 0x422800** (본 문서의 모든
  "엔트리 오프셋"은 이 값을 더해 ISO 절대 오프셋으로 환산).

## BND3 아카이브 (DATA.BIN 상단)

```
+0x00  magic   "BND307H1V19" (버전 문자열 포함)
+0x20  TOC     16바이트 레코드 × 10,344개
        u32 flags
        u32 size
        u32 offset      (DATA.BIN 상대)
        u32 id          (논리 ID — 종류별 대역 구분)
```

논리 ID 대역(관찰):
- `1200000~1299999` : 미션 HUD/캡션 폰트 등 미션 공용 자원
- `4000000~4002999` : 부팅/프론트엔드(셸) 자원
- `600x`(idx 180~ 부근) : 미션별 텍스트/스크립트 섹션
- `700xxx`의 0x40000급 블롭 : SCEI VAG 음성(폰트 아님)

엔트리 내부가 다시 **inner BND**(`"BND\0"` magic)인 경우가 많다:

```
+0x00  "BND\0"
+0x04  u32 total_size
+0x08  u32 count
+0x18  (u32 fid, u32 offset) × count      — offset 오름차순
```

폰트 엔트리에서 자주 쓰는 inner fid: `2000`(대형 텍스처), `2500`(대형 폰트블록),
`2001`/`2501`(소형 변형). 경보 HUD 폰트(idx 4614)는 예외적으로 텍스처가 inner 목록에
없고 outer `+0x60`에 raw로 놓인다.

## 텍스트 컨테이너 (magic `00 01 00 00`)

```
+0x00  u32 magic = 0x10000 (LE)
+0x10  u32 count           (문자열 슬롯 수)
+0x14  u32 tbloff          (통상 0x28, 최대 0x1000까지 관찰)
+tbloff u32 offset × count (0 = 빈 슬롯; 컨테이너 base 상대)
...    null 종료 Shift-JIS 문자열 힙
```

- 한 BND 엔트리에 컨테이너가 여러 개 있을 수 있다.
- 힙은 원본 크기(`csize`)가 상한. 재조립 시 중복 문자열은 오프셋 공유로 압축 가능.
- 총 33,000+ 문자열, 고유 일본어 약 11,254개(약 53만 자).
- 문자열 파서 검증은 SJIS **구조 검증**(lead 0x81-0x9F/0xE0-0xFC + trail 0x40-0x7E/0x80-0xFC)
  기준 — 코덱 미정의 유효쌍(버튼 아이콘 등)을 통과시키기 위함.

### 인라인 태그

`<ab(800)>`, `<op()>`, `<sp(20)>`, `<on()>`, `<color...>` 등 ASCII 앵글 태그 34,800여 개와
`#c[RRGGBB]`, `⟦XXXX⟧`(raw 2바이트 이스케이프), `%d` 서식이 본문에 섞인다.
번역 시 태그 멀티셋과 개행 수를 원문과 일치시키는 하드 게이트가 필요하다.

### 런타임 문자 키

게임 내부 문자 키 = **SJIS 코드 + 0x4000** (런타임 테이블·미션 서브블록 키에서 공통 확인).

## 기타

- 미션 서브블록(id X451 등): magic `0x100` 아카이브, 섹션 오프셋 헤더, (SJIS+0x4000) 키 테이블 내장.
- 화자명 리스트: 00010000 컨테이너와 **매직 없는 raw NUL 구분 연속 리스트**(포인터 없음)의 이중 존재.
