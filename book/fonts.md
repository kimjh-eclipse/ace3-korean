# 폰트 시스템 지도

게임에는 용도가 다른 폰트가 **14종+** 존재하며, 넷 이상의 서로 다른 렌더러가 이를 소비한다.

## 폰트 목록

| 구분 | 엔트리 | 용도 | 비고 |
|---|---|---|---|
| 메인 폰트 | BND idx4(맵)/idx5(텍스처) | (초기 가설과 달리) 인게임 대사 렌더에 직접 쓰이지 않음 | 1024x512 4bpp linear |
| 셸 폰트 6종 | idx8780/81/82/83/84/87 (id4002050~57) | 타이틀·세이브/로드·메뉴·인터미션 등 프론트엔드 | 폰트별 charset 상이(1393~1447) |
| 캡션(무전자막) 대형 6종 | idx4603/04/07/08/09/10 (id1200000~7) | 미션 무전 자막·목표 배너·이벤트 컷신 대사 | charset 1428, GS덤프로 사용 확정 |
| 캡션 소형 2종 | idx4605/06 (id1200002~3) | 일부 미션의 소형 자막 | 512x512, charset 698 |
| 경보 HUD 폰트 | idx4614 (id1200011) | "에어리어 오버" 등 중앙 경보 배너 | 텍스처가 outer+0x60 raw(변형 구조) |

## 공통 구조 (셸/캡션 계열)

```
[스위즐 텍스처 0x40000] [16색 RGBA 팔레트 0x40] [폰트블록]
```

inner BND 기준: fid 2000 = 텍스처 엔트리(+0x20부터 payload 0x40000, 끝 0x40 팔레트),
fid 2500 = 폰트블록.

### 폰트블록

```
+0x10  u32 packed = (nglyph << 16) | nranges
+0x18  u32 metric_off            (폰트블록 상대)
+0x20  range 테이블: (u32 a, u32 z, u32 glyph_idx) × nranges
        — 코드 a..z 가 glyph_idx.. 에 연속 매핑. 시작코드 기준 거의 오름차순.
+metric_off  글리프당 24바이트 × nglyph:
        f32 u0, v0, u1, v1      (정규화 UV — 아틀라스 박스)
        u16 xofs, w, adv, 0     (관찰상 대부분 xofs=0, w=adv=박스폭)
```

### 텍스처 스위즐

1024x512(또는 512x512) 4bpp를 **"PSMCT32로 업로드된 PSMT4"** GS 스위즐로 저장.
순수 linear도 순수 PSMT4도 아니다. PCSX2 GSTables 기반 변환 테이블로
디스위즐/재스위즐하며 바이트 라운드트립이 검증되어 있다(`deswz_tables.py`,
`build_file_index_4via32(W,H)`).

GS 업로드 형태: `BITBLTBUF.DPSM=PSMCT32, DBW=8, TRXREG=512x128` (= 0x40000 바이트).

### 팔레트 (핵심 함정)

| idx | 색 | 알파 |
|---|---|---|
| 0 | 배경 | **0 (투명)** |
| 1 | 0x202020 (어두운 외곽선) | 0x80 |
| 2~7 | 중간조 | 0x80 |
| 8 | 0xB4B4B4 (본체, 흰색 계열) | 0x80 |
| 9 | 어두운 중간조 | 0x80 |
| 10~15 | 검정 | **0 (투명!)** |

**픽셀값 15로 그리면 인게임에서 투명이 된다** — 본체는 반드시 8, 외곽선은 1로 그릴 것.
원본 글리프의 잉크 여백은 박스 안에서 상하좌우 최소 1px(대다수 1,1,1,1).

### 원본 글리프 규격 (19px 박스 기준 실측)

- 반각 숫자/영문: 잉크 2~15행(총 14px), 본체 3~14행(12px)
- 한자/가나: 잉크 1~16행(16~17px)
- 자간(adv)은 메트릭 u16이 아니라 박스폭과 사실상 동일
