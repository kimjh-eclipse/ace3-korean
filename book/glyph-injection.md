# 한글 글리프 주입 기법

## donor(기증) 코드 방식

폰트 구조를 바꿀 수 없으므로(엔진 제약), **원본 charset에 이미 있는 한자/가나 코드** 중
게임이 쓰지 않는 칸을 한글 음절에 1:1로 빌려준다.

- 맵: `kr_map_shell.json` — 한글 음절 1,188개 → donor 2바이트 SJIS 코드.
- 모든 텍스트를 이 맵으로 인코딩(2B→2B, 크기 불변)하고, 폰트 13종+의
  해당 박스 텍스처에만 한글 글리프를 덮어그린다. 어떤 렌더러(baked/파일 테이블)든
  같은 코드에서 같은 박스를 보므로 범용으로 동작한다.

### donor 선정 랭킹의 함정

donor 박스 크기는 **폰트 변형마다 다르다.** 소형 변형(inner 2501)에서 넓은 코드가
실제 캡션(inner 2500, 1024x512)에서는 13~15px로 좁을 수 있다(가나 영역).
반드시 **캡션 2500 기준 폭≥16·높이≥18**을 랭킹 필수 조건으로 걸 것.
(이걸 놓치면 고빈도 음절이 좁은 칸에 배정돼 찌그러진다 — 실제 사고 사례:
합/했/래/바/동 등 12음절.)

## 글리프 렌더링: 반드시 내장 비트맵(mono)

- 벡터 폰트를 14px로 안티앨리어싱 후 임계값 이진화하면 **가는 획이 소실**된다
  (ㅇ의 왼쪽 곡선 등). 품질도 음절마다 들쭉날쭉해진다.
- **`ImageFont.getmask(ch, mode="1")`** 로 batang.ttc의 **수제 내장 비트맵 글리프**를
  그대로 받으면 픽셀 균일·획 손실 0. 이것이 원본 게임 글리프와 같은 제작 방식이다.

## 배치 규격 (원본과의 통일)

| 항목 | 값 | 근거 |
|---|---|---|
| 크기 | 전 음절 **균일 14px** | 원본 반각 영숫자 본체 12px·전체 14px와 높이 일치 |
| 세로 정렬 | 잉크 바닥 = **15행** (19px 박스) | 원본 숫자의 잉크 바닥행과 베이스라인 공유 |
| 잉크 여백 | 상하좌우 ≥1px | 바이리니어 이웃 번짐 방지(원본 규칙과 동일) |
| 본체/외곽 | idx **8** / idx **1** | 팔레트 규칙(15는 투명!) |
| 외곽선(halo) | **박스 가장자리까지 허용** | 잉크가 여백 경계에 닿는 글리프(ㅇ의 좌측 호)도 사방 외곽선 필요 — 없으면 밝은 배경에서 가장자리가 사라져 "잘려 보임". idx1은 50% 알파 암색이라 이웃 번짐 무시 가능 |

## 공백 처리

원문(일본어)에는 어절 공백이 없어, 한국어 공백은 순수 추가 비용이다.

| 방식 | 결과 |
|---|---|
| 전각 공백 0x8140 (2B) | 라디오 섹션 402곳 컨테이너 오버플로 → 불가 |
| 0x20 | 엔진이 렌더하지 않음(？) → 불가 |
| range 추가로 새 공백 코드 | 엔진이 발견 못 함/크래시 → 불가 |
| **채택**: charset에 이미 있는 미사용 1바이트 코드 **0x5F('_')를 가짜 공백으로** | 글리프 박스를 blank 처리, 인코더가 공백을 0x5F로 emit. 오버플로 4곳으로 감소 |

단 **경보 폰트(id1200011)는 charset에 0x5F가 없다** → 문자열별 분기:
무전 태그(`<op()>`/`<sp(20)>`/`<on()>`)가 있는 대사는 0x5F, 태그 없는 배너/알림류는
전각 공백 0x8140(어느 폰트에서나 공백)으로 인코딩한다.

## charset에 없는 음절 (사례: '장')

캡션 charset에 없는 donor 코드가 배정된 음절은 자막에서 `？`가 된다.
range를 추가할 수 없으므로 **텍스트 레벨 치환**으로 우회한다:
charset에 존재하고 어떤 문자열도 emit하지 않는 코드(예: 0x88A3)를 골라
그 박스에 해당 음절을 그리고, 인코더가 그 음절을 그 코드로 emit한다.
(donor 랭킹을 캡션 기준으로 고친 뒤에는 1188/1188 전 음절이 커버되어
이 우회는 안전망으로만 남아 있다.)

## 검증 체크리스트

1. 여백 스캔: 전 글리프 잉크 여백 ≥1px (위반 0이어야 함)
2. 팔레트 시뮬레이션 렌더(알파 반영, 밝은 배경 합성)로 외곽선/본체 확인
3. 인코딩 스팟체크: 공백 바이트, 특수 치환 코드 hex 확인
4. 오버플로/폴백 카운트 0 확인
5. PCSX2 실기 + 필요시 GS 덤프 역추적
